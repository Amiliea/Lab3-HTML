*Лабораторная работа №3*
1. **Почему выбрали этот препроцессор?**

Я остановилась на SCSS(вариант sass), потому что он имеет CSS-подобный синтаксис - удобно писать, как в обычном CSS, но с мощными фичами вроде перемнных, вложенности и миксинов. Это идеально для профиля: легко масштабировать цвета, стили и адаптивность. SCSS популярен (в Bootstrap, Angular), компилируется просто через VS Code или npm, и превосходит LESS по функциям (loops, maps), а Stylus - слишком нишевый. В итоге, код вышел модульным и читаемым.
2. **Какие файлы для компонентов препроцессора создавали?**

Структура в styles/ получилась модульной: частные файлы импортируются в главный, чтобы не компилировать зря.
- _variables.scss: Глобальные переменные - $main-color: #4CAF50;, $avatar-size: 100px;, $font-family: 'Arial', sans-serif. Плюс наборы для тем: $light-background-color: #f8f8f8;, $dark-background-color: #121212;, $light-button-bg: #007bff; и т.д.
- _mixins.scss: Миксины для переиспользования - @mixin media-query($breakpoint) для адаптивности и @mixin theme-colors($theme: light) для тем (if/else с цветами).
- style.scss: Основной — импорты, вложенность (.profile { &__header { ... } }), grid и вызовы миксинов. Компилируется в style.css.
  Это позволило легко добавлять элементы вроде bio или skills без дублирования.
3. **Каким образом реализована тёмная тема страницы?**

Тёмная тема — через Sass-переменные и миксин, без CSS custom props, чтобы всё в препроцессоре. В _variables.scss — отдельные наборы цветов (light/dark для фона, текста, карточки, кнопки). Миксин @mixin theme-colors($theme) применяет их условно: @if $theme == dark { body { background-color: $dark-background-color; } .profile { background-color: $dark-card-bg; } &__button { background-color: $dark-button-bg; } } (с hover через darken()). В style.scss: базовая light, а для dark — body.dark-theme { @include theme-colors(dark); }. В HTML: <body class="dark-theme"> — добавь/убери класс, сохрани — тема переключается. Работает с grid и hover, компилируется в чистый CSS.
